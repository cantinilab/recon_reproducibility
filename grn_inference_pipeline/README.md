# GRN Inference based on the GRETA benchmark

This repository contains the benchmark for Gene Regulatory Network (GRN) inference using the GRETA pipeline:
[https://github.com/saezlab/greta](https://github.com/saezlab/greta)

The workflow is implemented with `Snakemake` to ensure full reproducibility of the analyses.

---

## Installation

### Install mamba

We recommend using `mamba` for faster and more reliable package installation.

```bash
conda install -n base -c conda-forge mamba
```

### Create a Snakemake environment

Create and activate a dedicated environment for running the pipeline:

```bash
mamba create -c conda-forge -c bioconda -n snakemake snakemake
conda activate snakemake
```

---

## Install Singularity / Apptainer

The pipeline relies on containerized software to ensure reproducibility.

### Linux installation

Install Apptainer following the official documentation:
[https://apptainer.org/docs/admin/main/installation.html](https://apptainer.org/docs/admin/main/installation.html)

On many HPC systems, Apptainer is already available as a module:

```bash
module load apptainer
```

### Verify installation

```bash
apptainer --version
```

## Obtain the GRNs used in ReCoN

The GRNs used in the ReCoN publication were generated using **HuMMuS**.

⚠️ **Important note on reproducibility**
GRN inference relies on stochastic steps in both the scATAC-seq and scRNA-seq layers. As a result, networks generated by rerunning the pipeline may differ slightly from those reported in the publication.

If you require the **exact GRNs used in the ReCoN manuscript**, please download them from the associated Zenodo record:
[https://doi.org/10.5281/zenodo.XXXXXXX](https://doi.org/10.5281/zenodo.XXXXXXX)

### Download the Singularity image (HuMMuS)

The HuMMuS pipeline relies on a Singularity/Apptainer image (`hummus.sif`). You can either **download the pre-built image from Zenodo** (recommended for full reproducibility) or **build it locally** from the provided definition file.

#### Option 1: Download the pre-built image from Zenodo (recommended)

Download the `hummus.sif` image associated with the ReCoN publication from Zenodo:

```bash
wget https://zenodo.org/record/XXXXXXX/files/hummus.sif
```

Place the image in a directory accessible to Snakemake (for example, `envs/` or a shared container cache), and make sure Snakemake can locate it via the `--use-singularity` flag.

#### Option 2: Build the image locally

If you prefer to build the container yourself, you can do so using the provided Singularity definition file:

```bash
singularity build envs/hummus.sif envs/hummus.def
```

⚠️ Building the image requires root privileges or a system configured for unprivileged builds. On most HPC systems, this step should be performed on a dedicated build node or locally.

---

### Generate GRNs from scratch

Clone the HuMMuS repository:

```bash
git clone https://github.com/cantinilab/recon_reproducibility.git
cd hummus
```

If running on an HPC system, load Apptainer/Singularity:

```bash
module load apptainer
```

#### Immune Dictionary dataset (spleen hybrid)

```bash
snakemake -c 50 \
  ./datasets/spleen_hybrid/cases/all/runs/hummus.hummus.hummus.hummus.mdl.csv \
  --use-singularity
```

#### Heart-related showcases (Heart Atlas)

```bash
snakemake -c 50 \
  ./datasets/heartatlas/cases/all/runs/hummus.hummus.hummus.hummus.mdl.csv \
  --use-singularity
```

## Reproducibility notes

* All software dependencies are resolved via Snakemake and Singularity containers.
* Exact versions used in the publication are preserved in the Zenodo archive.
* Computational requirements depend on dataset size; we recommend running with ≥32 cores for large datasets.
